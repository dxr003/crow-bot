# 2026-04-19 做多阻击 C维度 v4.0 重构：量比倍数 → 动能加速度

## 背景

乌鸦审视 MYX/NOM 炸单根因：C=10分依赖"量比倍数"，但倍数对冷币基线极度敏感——MYX 52.1x 是"过去55m平均2万/分钟 × 擦线103万突破"的冷基线刺激，不是真爆发。

更深层问题：**量比"绝对值"本身是滞后指标——等我们观察到大量买入，价格已经被拉起来，开仓 = 买在这波拉升尾部**。

## 乌鸦洞察链

1. "高倍数=真信号"是假的：MYX 52x → 炸；SIREN 2.5x 才是真热门
2. 砸盘也会放量，k[10]主动买入是对的切入点
3. 量的"绝对值"是滞后指标，量的"加速度"才是动能
4. 5分钟窗口只抓"闪电爆发"，漏掉"阶梯式慢涨"
5. 单笔数量 + 买卖正比 是动能识别的第三/第二轴

## 方案：C维度 v4.0 三轴动能评分

**数据轴**（fetch_market_data 新增字段）：
- `short_accel` = 最近2m买入/分钟 ÷ 前3m买入/分钟（闪电爆发检测）
- `mid_accel` = 最近5m买入/分钟 ÷ 前10m买入/分钟（阶梯慢涨检测）
- `buy_ratio_5m` = 最近5m主动买入 ÷ 最近5m总成交（多空对决）
- `back_2m_buy_usdt` = 最近2m主动买入绝对额（冷币过滤）

**前置闸门**（任一不过 → C=0）：
- 买占比 ≥ 55%（防砸盘假象）
- 最近2m主动买入 ≥ 20万U（防冷币虚高加速度）

**评分阶梯**（互斥取最高）：
| 场景 | 短期加速 | 中期加速 | 得分 |
|---|---|---|---|
| 双共振爆发 | ≥3x | ≥3x | 10 |
| 接近双共振 | 一边≥3x 另一边≥2x | | 8 |
| 双共振强势 | ≥2x | ≥2x | 7 |
| 单维度爆发 | 任一≥3x | | 5 |
| 短强中温/短温中强 | 一边≥2x 另一边≥1.5x | | 4 |
| 温和启动 | 任一≥1.5x | | 2 |
| 平稳 | 1-1.5x | 1-1.5x | 0 |
| 衰减 | 任一<1x | | -3 |

## 改动清单

### scanner.py
- `fetch_market_data()` 扩展：拉15根1m K线切分为 [最近2m, 2-5m, 5-15m] 区间
- 新增输出字段：`short_accel` / `mid_accel` / `buy_ratio_5m` / `back_2m_buy_usdt`
- 旧字段 `volume_ratio` / `vol_5m_buy_usdt` 保留（供进池逻辑和卡片显示）
- K线不足15根时降级走旧算法（只有进池初期几秒会短暂降级）

### analyzer.py
- C维度评分彻底重写，从"volume_ratio 倍数档位"改为"加速度+买占比+绝对量三轴"
- 闸门在前，评分在后，breakdown 标签明确说明闸门否/场景分类
- 旧的 `vol_ratio_min_abs_5m` 封顶逻辑删除（新架构天然含绝对量底线）

### config.yaml
- 删除：`vol_ratio_5x/3x/2x/1_5x/1x/low`、`vol_ratio_min_abs_5m`
- 新增：`c_buy_ratio_gate: 0.55`、`c_min_back_2m_usdt: 200000`
- 新增评分参数：`c_dual_super: 10` / `c_near_dual: 8` / `c_dual_strong: 7` / `c_one_super: 5` / `c_mixed: 4` / `c_mild: 2` / `c_flat: 0` / `c_decay: -3`

## 实测验证

重启前用真实数据跑 SIREN（刚过甜蜜区的币）：
- short_accel=0.58, mid_accel=0.76（衰减中）
- buy_ratio=48.5%（<55%）
- back_2m_buy=9.5万U（<20万）
- **结果：C=0 闸门否（"买占比48%<55%闸门否"）**，总分16不过线 ✓

旧逻辑下 SIREN 之前会拿到 C=10（量比高+5m买入34万过封顶线），新逻辑直接识别出"量能在收缩"。

## 保留的决策点

- **"底部预备加速布局"信号（价格没动但量在温和放量）暂不单独加档**：当前架构能抓到"接近双共振8分"档，需配合A+B+D+E凑到30分才开仓。先观察1-2周数据，看是否错过关键信号。
- **单笔大单识别（aggTrades）第二期再上**：当前三轴已经能打死 MYX/NOM 型追高信号，单笔识别属于"再精细一层"，先观察三轴效果。

## 不受影响

- 进池逻辑（pool_vol_ratio_min=2.0 / pool_stair_vol_ratio=1.5）：用 `calc_volume_ratio()` 函数，与C维度解耦，保留不动
- 其他维度（A价格动能 / B池内涨幅 / D OI+费率 / E链上聪明钱 / G公告）：全部不动
- signal_threshold=30 保留
