# 2026-04-17 变更日志

## 做多阻击 v3.3 里程碑

### 核心修复

#### 1. OI评分维度修复（D维度）
- **问题**：get_oi_change() 用内存缓存做基准，30秒间隔变化率几乎为0，D维度从未生效
- **修复**：改用币安历史OI API `/futures/data/openInterestHist`，滑动窗口对比
- **配置**：`oi_compare_period: "15m"`（支持5m/15m/30m/1h/2h/4h，热重载）
- **效果**：修复后立即生效，PNUT +3.5%→+3分，TRB +6.7%→+5分
- 文件：scanner.py（封板）

#### 2. reject_tracker 修复（两层bug）
- **Bug1**：`changed = True` 只在价格超过峰值时设置，价格下跌时samples更新但不保存
- **修复**：移到 `if price > 0` 内部，任何价格更新都触发保存
- **Bug2**：scanner构建_reject_prices用 `t.get("lastPrice", 0)` 但tickers用 "price" key
- **修复**：两处改为 `t.get("price", 0)`
- 文件：reject_tracker.py, scanner.py

#### 3. score_history 字段名修复
- `vol_ratio` → 匹配 `market_data["volume_ratio"]`
- `oi_usdt` → 匹配 `market_data["oi_change_pct"]`
- 文件：scanner.py

### 数据分析发现

#### 最低进场涨幅分析（基于2417条score_history）
- 0-2% 涨幅：仅1.3%记录能过28分（需要E因子逆天）
- 2-5%：5.2%
- 5-10%：9.6%（微调后预计提升到15%+）
- 10-15%：16.3%（主力出信号区间）
- 结论：系统实际需要5-10%以上涨幅才有合理概率触发买入

- AXL：评分33但24h涨51.6%被拦，即使买入也无法止盈（正确拦截）
- 系统无遗漏，未出信号是因为确实没有达标的妖币
