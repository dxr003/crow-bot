## 1. 策略复述（v1 保留，2.9 条目由乌鸦修正）

**一句话**：在一个被验证过的真箱体里来回做"贴沿入场 → 动态追踪止盈 → 浮亏扛单加仓 → 破箱才退场"，止盈像水龙头开小阀慢慢收，止损只有一个唯一条件是箱体自己死掉。

### 1.1 哲学骨架

策略不预测方向，也不赌反转。它**只做一件事**：利用"箱体内价格被上下沿反弹回来"的统计规律，在两端反复开反向仓位吃小波，用保证金厚度消除"被单次插针打爆"的风险。爆仓的可能性从概念上被挪到了"箱体死亡"这个更大尺度的事件上——只要箱体还活，浮亏就不是问题，甚至是机会（加仓拉均价）。

### 1.2 五状态机（玩家视角）

| 状态 | 心理状态 | 下一步 |
|---|---|---|
| 1 无持仓 | "等它贴边" | 贴下沿开多 / 贴上沿开空 / 中间等 |
| 2a 浮盈未激活 | "小赚但不够" | 观察破箱 + 观察是否该加仓 |
| 2b 浮盈追踪 | "盯 peak 收水龙头" | peak 回撤 30% 平仓 / 破箱全平 |
| 3 浮亏 | "扛着，让他跌" | 浮亏到新均价 1.5% 再加一层 / 破箱就全平 |
| 4 刚平仓 | "看位置决定下一秒" | 冷却 10s 后：贴对岸反手 / 贴原方向复开 / 中间归零等 |
| 5 破箱冷却 | "出事了，停 6 小时" | 到时间重新评估箱体 → 恢复就归状态 1 |

WebSocket 毫秒驱动状态转换；REST 补充 K 线 / OI / 费率。

### 1.3 止盈本质

不是"到 $10 就跑"，是"到 $10 把水龙头转小 30%"。$10 激活后盯 peak，只要不从 peak 滑 30% 就继续陪涨。收益自适应：小波 $7、中波 $17、大波 $41，同一套规则。

### 1.4 止损本质 · 2.9 乌鸦修正版 ⚠️

**唯一止损 = 箱体死了。和持仓方向无关、和浮盈浮亏无关。**

v1 我写过"浮盈时破向对岸本质是你赚的方向更强"，这是**理解偏了**。

破箱 = **箱体形态死亡** = **策略前提消失** = **下一步完全不可预测**。

举例：持多仓均价 $70,000，价格破 upper $82,000，浮盈 $6,000。
- 这 $6,000 是**过期假设下建的仓**，箱体这个统计规律已经失效
- 接下来可能继续涨到 $90,000（赚更多），也可能插针回 $75,000（吐回去）
- 在"前提不成立"的行情里继续持仓 = 赌博，不是策略

**所以破箱判定只看 4/4 共振：满足就全平，不看你是多是空、赚还是亏、破哪一侧。trailing 不继续跑。** 这条原则反直觉，但是策略视角的正确心法——前提消失立刻离场。

### 1.5 AI 哨兵

三个位置：启动前验箱体真伪 / 破箱 3/4 临界裁决 / 健康度下降提醒提前退。日 <50 次调用 / <$1，haiku-4-5。

### 1.6 事件账本

11 个 event_type（entered_long/short、activation_reached、trailing_updated、small_profit_taken、added_position、reversed、re_entered、breach_flat、cooldown_started/ended、ai_consulted）。payload 固定含 `strategy=gridrunner` / `instance_id` / `position_pct` / 浮盈状态 / 相关指标。需加进 conventions v1.2（乌鸦让先别动，待批）。

### 1.7 与 bull_sniper 关系

完全独立。共用底层（exchange/order/risk/ledger/websocket），不共享策略逻辑、参数、状态。同币种可双向共存，冲突由账户级 risk.py 兜底。

## 2. v2 已明确的规则（乌鸦补充/修正 10 条）

下面是 v1 疑问点的最终答复，直接作为 P1 实装依据。

### 2.1 仓位尺寸

- `base_size_usd = $500`（默认）
- `max_layers = 5`
- **累计最大 = $500 × 5 = $2,500**（替代原 max_total_usd 默认 3000，实际以层数公式为准）

### 2.2 反手冷却闸门（v1 建议采纳）

状态 4 的反手 / 复开受双约束：

- **时间闸门**：平仓到反手至少 **10 秒**
- **收益闸门**：反手的激活阈值 = `activation_usd × 2`
  - 即本轮必须先赚到 ≥ $20 才允许反手开仓
  - 小波反复 → 手续费吃收益 → 这道闸门直接切断

### 2.3 浮亏加仓基准

**相对当前均价**（v1 建议方向正确）。

- 每加一层后均价更新
- 下次加仓触发线基于**新均价** × (1 - 1.5%)
- 天然收敛：均价逐层下移，触发点也往低走，**不会无休止加仓**

### 2.4 保证金安全垫（v1 发现的 bug 修正）

双保险：

- **公式层**：`safety_ratio` 从 1.3 升到 **1.5**
- **数值层**：启动前加"5 层全建完 + 价格扫到 lower-zone 外侧 1.5%"的工况模拟，算实际浮亏是否 < min_margin
- **方向差异**：做空时 `safety_ratio ≥ 1.5`（上破更频繁，经验性上调）

### 2.5 破箱 4 因子时间窗口

| 因子 | 窗口 |
|---|---|
| 1 · 价格突破 upper/lower 超过 1.5% | 实时（tick） |
| 2 · 成交量 > 过去均值 × 2 | **过去 20 根 15 分钟 K 线（5 小时窗口）** |
| 3 · OI 变化 > 3% | **15 分钟窗口** |
| 4 · 资金费率方向与突破一致 | **最近两期（16 小时窗口）** |

### 2.6 破箱判定和方向/浮盈完全无关

见 1.4。**状态 2b/3 的破箱逻辑完全一致，不区分浮盈浮亏**。4/4 满足全平，3/4 问 AI，<3/4 当假突破。

### 2.7 多实例部署

**4 个实例各跑一品种**（v1 推荐方向）：

- BTC / ETH / SOL / BNB 各自独立 `gridrunner_{symbol}.yaml`
- 各自独立状态机、独立 instance_id（例 `btc_001`, `eth_001`）
- 资源统筹（保证金分配、WebSocket 订阅、AI 预算）在**上层管理器**

### 2.8 账户绑定

**专用账户跑 gridrunner**，和 bull_sniper / 玄玄手动交易物理隔离。

- 不走 `positionSide` 区分、直接账户分离
- 新账户（具体账号待乌鸦定）或从现有 4 账户里划一个专属

### 2.9 连续破箱兜底（v1 建议 3.5 采纳）

- **连续 3 次冷却失败 → 实例自动停用**
- 发 TG 告警等乌鸦手动重启
- 避免在僵尸箱体上无限空转

### 2.10 箱体恢复后的处理

冷却期结束重评时：

- **AI 评估"是否同一箱体"**
- 形态稳定 → 保留原 upper/lower
- 形态变化 → 重新识别上下沿（进入新一轮箱体）

## 3. 工程实现层约定（策略层不写死，实装时定）

下面是 v1 的 2.6 / 2.10 / 2.11 在实装层面的处理，乌鸦已给方向：

### 3.1 毫秒级触发 vs REST 下单延迟（v1 · 2.6）

- **浮盈计算用 mark_price**（不是 ticker 中间价），更平滑不跳
- **触发防抖：连续 2 tick 确认**才下单，避免单 tick 抖动误触发
- 平仓语义：用防抖后的判决信号 → reduce-only 市价单 → 实际成交价作为"实现浮盈"写入事件

### 3.2 健康度评分（v1 · 2.10）

- **本地函数评分**为主（触碰频次 / 突破次数 / 均价漂移等）
- **AI 在阈值下时补判**：本地分 < 某阈值才调 haiku，避免每次都问
- 日调用预算仍控制在 <50 次

### 3.3 触碰定义（v1 · 2.11）

- **"进 zone 算一次触碰"**（非点到点）
- **连续波动算一次**（价格在 zone 内反复不算多次，离开 zone 后再进才算下一次）
- K 线周期：与 2.5 对齐，用 15m 统一

### 3.4 position_pct 强制定义（v1 建议采纳）

所有事件 payload 的 `position_pct` 统一为：

```
position_pct = (当前价 - lower) / (upper - lower) × 100
```

0 = 贴 lower，100 = 贴 upper，>100 = 破上沿，<0 = 破下沿。
加进 conventions v1.2（待乌鸦批准动文档）。

## 4. 保留的设计边界

### 4.1 v2 自适应上下沿（v1 · 3.6 采纳）

**暂不启用**。条件：

- v1 跑满 **30 天**
- 在多种箱体形态（窄箱/宽箱/倾斜箱）测过
- v2 启用前必须先写清楚"只在无持仓 / 冷却期改 upper/lower"的约束

运行中修上下沿会让 peak 基准、加仓触发线、保证金公式全部漂移，必须无持仓时才能动。

### 4.2 kill-switch 层级

| 层级 | 触发 | 动作 |
|---|---|---|
| 单次破箱 | 4/4 共振 | 全平 → 6 小时冷却 |
| 连续破箱 | 3 次冷却失败 | 实例停用 → TG 告警 |
| 全策略 | 账户级 risk.py 拦截 | 所有实例停手 |
