# 做空阻击系统 — 信号策略定稿

> 定稿日期：2026-04-07
> 原名：做空猎手 v2.0，重命名为做空阻击系统

---

## 一、系统定位

扫描币安合约暴涨榜，捕捉短线拉盘见顶后的做空机会。
**只推送信号，开空由乌鸦决定（后续可对接自动下单）。**

---

## 二、两层信号结构

### 第一层：预警播报（信息层）
低门槛，广播用，不带开仓建议。

### 第二层：做空信号（执行层）
高门槛，精准信号，带强平价开仓建议。

**两层完全独立，不混淆。**

---

## 三、状态机

```
扫描发现(≥80%) → 监控中 → 触发做空信号(150%+回撤20%) → 持仓中 → 击中/失败 → 退出
                    ↓
              跌回60%以下 → 退出监控
```

### 状态说明

| 状态 | 进入条件 | 退出条件 |
|------|---------|---------|
| 监控中 | 24h涨幅 ≥80% | 跌回60%以下 / 超72小时 |
| 持仓中 | 从发现价上涨≥150% 且从最高价回撤≥20% | 击中 / 失败 |
| 击中✅ | 从买入价下跌≥50% | — |
| 失败❌ | 价格超过强平价 | — |

---

## 四、核心判断逻辑

**基准价格：** 第一次发现时的价格 = `price_at_entry`（归零点）
后续所有计算基于此价格，不再看24h滚动涨幅。

| 判断项 | 公式 |
|--------|------|
| 是否进入监控 | 24h涨幅 ≥80% |
| 是否触发信号 | (max_price - price_at_entry) / price_at_entry ≥150% 且 (max_price - cur_price) / max_price ≥20% |
| 击中判断 | cur_price ≤ position_price × 50% |
| 失败判断 | cur_price ≥ 强平价 |
| 自然退出 | cur_price ≤ price_at_entry × (1 - 20%) 即跌回发现价60%以下 |

---

## 五、开仓建议（强平价反推）

做空信号触发时，系统推荐：
- **强平价 = 信号触发时市价 × 120%**
- 使用强平价反推公式开单（qty = margin / (liq × (1+MMR) - entry)）
- 理论上币需要再涨约100%才爆仓，安全边际大
- 币涨得越高开空胜率越大，强平价随之抬高

> 强平价是系统预设建议值，乌鸦可自行调整。超过强平价记为"失败"，不代表真实爆仓（可私下加仓扛）。

## 六、推送机制

| 事件 | 推送方式 | 时机 |
|------|---------|------|
| 新币进入监控 | 立即推通知 | 随时，不等整点 |
| 监控→持仓信号触发 | 立即推醒目通知 | 随时，不等整点 |
| 定时状态卡片 | 每60分钟推一张完整卡片 | 整点 |
| 退出监控 | 合并进下一张定时卡片 | 不单独推 |

**推送目标：** 群组 @草币社区抱团小队

## 八、可配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `monitor_threshold` | 80% | 进入监控的涨幅阈值 |
| `signal_rise` | 150% | 从发现价上涨多少触发信号 |
| `signal_pullback` | 20% | 从最高价回撤多少触发信号 |
| `exit_threshold` | 60% | 跌回发现价多少以下退出监控 |
| `success_drop` | 50% | 从买入价跌多少算击中 |
| `liq_multiplier` | 120% | 强平价预设倍数（市价×120%） |
| `reentry_cooldown_h` | 24 | 退出后多少小时可重新监控同一币 |
| `monitor_expire_h` | 72 | 监控超时自动退出 |
| `scan_interval_min` | 5 | 扫描间隔（分钟） |
| `card_interval_min` | 60 | 定时卡片间隔（分钟） |

## 九、辅助数据（展示，不过滤，不评分）

持仓信号推送时附带显示，乌鸦自行判断，系统不做任何过滤或评分。

| 数据 | 说明 |
|------|------|
| 资金费率 | 展示当前数值 |
| OI变化 | 展示涨跌方向和幅度 |
| 24h成交量 | 展示数值 |

> 加评分系统会引入噪音，过滤条件反而降低胜率。策略本质是概率游戏：成功次数×平均收益 > 失败次数×平均亏损，赔率够大不需要高胜率。

## 十、与交易执行系统的关系

```
做空阻击系统（本模块）
    ↓ 推送信号
做空交易执行系统（独立模块，待开发）
    ├── 自动接收信号开仓
    └── 乌鸦手动输入开仓
```

两个系统解耦，做空阻击只管发信号，执行层独立开发。
