## 一、移动止盈策略 v2.0（定稿）

### 核心逻辑
追踪历史最高浮盈，从最高点回撤达阈值触发平仓。
回撤% = (历史最高浮盈 - 当前浮盈) / 历史最高浮盈 × 100%

### 三档预设
| 档位 | 指令 | 激活条件 |
|------|------|---------|
| 1档 | 移1 / 保 | 浮盈 ≥10% |
| 2档 | 移2 / 中（默认） | 浮盈 ≥30% |
| 3档 | 移3 / 激 | 浮盈 ≥60% |

### 动态回撤分层
| 浮盈区间 | 1档（保） | 2档（中） | 3档（激） |
|---------|---------|---------|---------|
| 激活～50% | 20% | 30% | 40% |
| 50～100% | 15% | 20% | 30% |
| 100～200% | 10% | 15% | 20% |
| 200%+ | 8% | 10% | 15% |

盈利越高刹车越灵敏，200%以上不再收紧，保留呼吸空间防止正常波动扫仓。

### 容错比（防抖）
±3%，防止正常波动误触发

## 二、滚仓策略 v2.0（定稿）

### 核心规则
| 参数 | 值 |
|------|-----|
| 触发条件 | 浮盈 ≥50% |
| 加仓金额 | 当前浮盈 × 70% |
| 容错比 | ±3% |

### 触发判断
- 浮盈 > 53% → 确认触发，执行加仓
- 浮盈 < 47% → 未触发
- 47%～53% → 观察，下轮再判断

### 执行示例
入场$1000 → 浮盈50%($500) → 加仓$350 → 新仓位$1350
新仓位浮盈再达53% → 再次触发（复利滚动）

无冷却时间，每次以新仓位为基准重新计算。

### 与移动止盈关系
独立设置，不绑定档位，并行运行。两者解耦，自由组合。

## 三、交易底座 v1.0（定稿）

### 整体流程
用户输入 → 解析层 → 标准JSON → 风控校验 → 预览卡片 → 用户确认(60s超时取消) → 交易所执行 → 回执卡片

### 指令格式（极简）
基础语序：动作 币 杠杆 金额 [附加参数]

示例：
- 多 sol 5 100
- 空 btc 10 50%
- 平 sol / 平 sol 50%
- 划 100 现货到合约
- 挂 sol 空 172 50u

附加参数（乱序识别）：
- 损3% / 损3000 → 止损
- 盈10% → 止盈
- 移1/2/3 → 移动止盈档位
- 滚 → 启动滚仓
- 拆5 → 拆5笔执行
- 暗单 10m → TWAP时间加权

### 风控校验层
| 检查项 | 规则 | 拦截方式 |
|--------|------|---------|
| 单笔上限 | ≤$50,000 | 硬拦截 |
| 日亏损 | ≤$10,000 | 硬拦截 |
| 总回撤 | ≤30% | 硬拦截 |
| 余额充足 | 可用余额 ≥ 保证金 | 硬拦截 |
| 合约状态 | 必须TRADING | 硬拦截 |
| 开反方向 | 与现有持仓方向相反 | 软提醒二次确认 |
| 重复开单 | 同币种同方向已有持仓 | 软提醒确认加仓 |
| 重复挂单 | 同币种同方向已有挂单 | 软提醒确认新增 |
| 重复移动止盈 | 同币种已在运行 | 软提醒确认替换 |
| 重复滚仓 | 同币种已在运行 | 软提醒确认重置 |

平仓时自动取消该币种所有关联挂单，回执显示"已取消X个挂单"。

### 解析层
优先硬解析（零延迟零token）→ 失败时AI兜底解析成标准JSON
